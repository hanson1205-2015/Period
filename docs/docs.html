Language Documentation

Every statement in Period ends with a period (.). Comments begin with two dashes and extend to the end of the line.

Installation & First Steps

On Windows, download the latest Period-Setup.exe from the download page and run it. The installer will:

  • Install period.exe to C:\Program Files\Period.
  • Add Period to your system PATH.
  • Install the VS Code extension automatically.

On Linux or macOS, build from source with Rust:

cd period
cargo build --release
sudo cp target/release/period /usr/local/bin/period

Period compiles every program to bytecode and runs it on a single virtual machine, so the semantics you see in the REPL are exactly the semantics you get in a file.

After installation, verify it works:

period --version

Run your first program:

period hello.period

Or start the interactive REPL:

period

Basics

A Period program is a sequence of statements. Each statement ends with a period. Whitespace and blank lines are ignored.

-- This is a comment.
let x be 10.
show x.

Keywords and reserved words (let, if, true, nothing, etc.) are case-insensitive: let, Let, and LET all mean the same thing. Variable and function names remain case-sensitive.

Variables

Use let to declare a variable and set to reassign an existing variable or index.

let name be "Period".
set name to "Period Language".

let items be [1, 2, 3].
set items[0] to 10.

Variable names must start with a letter or underscore and may contain letters, digits, and underscores.

Data Types

Numbers

Integers and floating-point numbers are supported. Arithmetic follows standard precedence rules. integer values use arbitrary-precision integers, so they are not limited to 64 bits and do not silently overflow. Literals containing a decimal point produce number (floating-point) values instead. Comparisons between integers and numbers use exact arithmetic whenever possible, so a large integer is not silently considered equal to a nearby floating-point value.

let count be 42.
let pi be 3.14.
let result be 2 ** 100.  -- arbitrary-precision integer
show result.

Strings

Strings are enclosed in double or single quotes (equivalent) and support \n, \t, \", \', and \\ escapes. You can interpolate expressions inside { }; each expression is automatically converted to its string representation. To include a literal brace, escape it as \{ or \}.

let name be "Period".
let lines be "First line\nSecond line".
show "Hello, {name}!".
show "1 + 2 = {1 + 2}".

Booleans and Nothing

true and false are the boolean values. nothing represents the absence of a value, similar to null in other languages.

let flag be true.
let empty be nothing.

Lists

Lists are ordered, mutable collections. Use brackets for indexing. Negative indices count from the end of the list.

let numbers be [1, 2, 3].
show numbers[0].
show numbers[-1].
set numbers[-1] to 10.
show length with numbers.

Dictionaries

Dictionaries store key-value pairs. Keys can be strings, numbers, or booleans.

let person be {"name": "Ada", "age": 36}.
show person["name"].
set person["age"] to 37.
show length with person.

Operators

Arithmetic

  • + addition (also string/list concatenation)
  • - subtraction and unary negation
  • * multiplication (also string repetition by an integer)
  • / division
  • % modulo
  • ** power

Comparison

==, !=, <, >, <=, >=

Logical

and, or, not

Operator Precedence

From highest to lowest: parentheses, indexing, unary not and -, **, * / %, + -, comparisons, and, or. The comma separating function-call arguments has the lowest precedence, so operators inside arguments are always evaluated first.

Conditionals

Use if ... then: to branch. The condition must be a boolean; Period has no implicit truthiness, so strings, numbers, lists, and dictionaries cannot be used directly as conditions. The optional otherwise: block runs when the condition is false; else: is an accepted alias.

let score be 85.
if score >= 60 then:
    show "pass".
otherwise:
    show "fail".

You can nest conditionals inside any block. Blocks end when the indentation returns to the previous level.

Loops

Use while with repeat: to loop while a condition is true.

let i be 0.
while i < 10 repeat:
    show i.
    set i to i + 1.

Use for ... in ... repeat: to iterate over a list, string, dictionary keys, or a range. The range built-in accepts one, two, or three integer arguments: range with <stop>, range with <start>, <stop>, or range with <start>, <stop>, <step>. Because ranges are iterated eagerly, their arguments must currently fit in a 64-bit signed integer; this is an implementation limit of range, not of the integer type itself.

for i in range with 5 repeat:
    show i.

for ch in "hello" repeat:
    show ch.

for n in [1, 2, 3] repeat:
    show n.

Functions

Define functions with define and call them with with. Use return to send a value back. Functions that do not explicitly return yield nothing.

define greet with name:
    show "Hello, " + name + "!".

greet with "World".

define factorial with n:
    if n <= 1 then:
        return 1.
    return n * factorial with (n - 1).

Function call arguments are full expressions separated by commas, so double with 3 + 4 means double(3 + 4) and add with 1 + 2, 3 + 4 means add(1 + 2, 3 + 4). Parentheses are only needed to group expressions differently or to disambiguate nested calls.

Parameters are passed by value. Functions close over the environment in which they are defined.

Compact Syntax

Period's English-style forms are great for readability, but they can become verbose for nested data or frequent calls. You can use the compact forms below wherever you prefer; they are fully equivalent to the English forms and can be mixed in the same program.

  • <expr>.<name> is the same as the <name> of <expr>.
  • <expr>.<method>(<args>) is the same as tell <expr> to <method> with <args>.
  • <expr>(<args>) is the same as <expr> with <args>.
  • new <Class>(<args>) is the same as new <Class> with <args>.
show sqrt(16).  -- same as: show sqrt with 16.

class Person:
    init with name, age:
        set this.name to name.
        set this.age to age.

    define greet with greeting:
        show greeting + ", " + this.name + "!".

let ada be new Person("Ada", 42).
ada.greet("Hello").  -- same as: tell ada to greet with "Hello".
show ada.age.          -- same as: show the age of ada.

Type Annotations

Type annotations are optional. When present, they are checked twice: the static checker validates what it can prove before execution, and the runtime re-checks every annotation, so dynamically-typed values can never slip past an annotation silently. Use them on parameters, return values, and variables. Function parameters read most naturally with the type first (number a); variable declarations read most naturally with the type after be (let count be integer 0).

define add with number a, number b returns number:
    return a + b.

define greet with string name returns string:
    return "Hello, " + name + "!".

let numbers be list of integer [1, 2, 3].
let scores be dictionary of string to number {"Ada": 90}.
let names be list of string ["Ada", "Grace"].

Built-in type names are nothing, boolean, integer, number, string, list, dictionary, function, class, and anything. integer values are a subset of number; an annotation of number accepts both integers and floating-point values. type with reports the more specific integer for whole-number literals. Compound types use list of <type> and dictionary of <key> to <value>. You can also use a class name or a generic type variable such as T to match instances of that class. anything is the gradual-typing escape hatch: it matches every value, and unannotated code is treated as anything statically while still being checked dynamically at runtime.

Several types can be combined into a union with or: number or string, or integer, number or string for three or more. A value matches a union when it matches any member.

define pick with number or string x returns number or string:
    return x.
let item be integer or string 42.

Classes

Define a class with class, an optional init constructor, and define methods. Inside methods, this refers to the current instance. You can use either the English-style forms or the compact dot forms for property access and method calls.

class Person:
    init with name, age:
        set this.name to name.
        set this.age to age.

    define greet with greeting:
        show greeting + ", " + this.name + "!".

let ada be new Person("Ada", 42).
ada.greet("Hello").
show ada.age.

Create instances with new <ClassName> with <args> or new <ClassName>(<args>), call methods with tell <object> to <method> with <args> or <object>.<method>(<args>), and access properties with the <property> of <object> or <object>.<property>.

set ada.age to 43.
show the age of ada.  -- English form still works

Fields can be declared either as init parameters or by assigning to the <name> of this or this.<name> inside init. The static type checker infers types for body-initialized fields and checks later property assignments; assigning a value of the wrong type is a static error. Methods are not properties: you cannot read a method with the <method> of <object> or <object>.<method>; doing so produces an error. Always call methods with tell or the compact <object>.<method>(...) form.

Imports

Use import to bring in built-in or standard-library modules. Multiple modules can be imported in one statement, separated by commas. The last separator may be replaced with and, but only one and is allowed.

import math.
import math, random, string.
import math, random and string.

After importing, call the exported functions directly by name:

import math, random and string.

let n be random.
let greeting be upper with "hello".
show sqrt with 2.

If two imported modules export the same name, you must qualify it with from:

import math, random and string.

show sin from math with 5.
let joined be join from string with "-", ["a", "b"].

Using an unqualified name that is exported by more than one imported module is an error.

Period ships with the following modules. They are imported by plain name, e.g. import list., and their exported signatures are checked statically:

  • math — trigonometry, roots, and rounding (native, .periodi stub)
  • random — random numbers; seed with <integer> re-seeds the generator so sequences become reproducible (native, .periodi stub)
  • string — case conversion, trim, split, search, replace, and slicing (native, .periodi stub)
  • time — timestamps (native, .periodi stub)
  • system — system integration: run with <command> (shell command, returns stdout), open with <target> (URL or file in the default handler), alert with <message> (message box), confirm with <message> (yes/no dialog, returns boolean), and notify with <title>, <message> (desktop notification). Dialogs and notifications use the native mechanism on each platform (Win32 MessageBox/WinRT toast on Windows, osascript on macOS, zenity/notify-send on Linux). (native, .periodi stub)
  • list — sum, max, min, map, filter, find, any, all, contains, reverse, slice, and sort (Period source)
  • text — text utilities (Period source)
  • path — minimal string path manipulation (Period source)
  • test — tiny assertion helpers for writing tests (Period source)

You can also write .periodi interface files (like Python .pyi stubs) that declare exports without implementation; the LSP reads them for completions and hover, while the runtime ignores them. Use ... as a placeholder expression or body. A leading string literal in a block is treated as a docstring and does not need a trailing .:

define sin with value returns number:
    "Return the sine of a value."
    ...

let pi be ....

Local modules must be imported with a POSIX-style relative path: ./helper loads helper.period from the same directory as the importing file, and ../utils/helper loads it from the parent directory's utils folder. A project lib/ folder is also checked for relative imports. Plain module names such as helper only resolve to built-in or standard-library modules.

Packages

Period has a built-in package manager. Packages are plain .period files published to a registry.

Using packages

Add a dependency to period.toml:

[package]
name = "my-project"
version = "0.1.0"

[dependencies]
list = "^1.0.0"
math = "~2.1.0"
hello = "=1.0.0"

Then run period install. The CLI downloads the registry index, resolves versions, fetches each package, verifies its SHA-256 checksum, and writes period.lock.

Version constraints support *, =1.2.3, ^1.2.3, and ~1.2.3. The registry index is a single JSON file hosted on a static site such as GitHub Pages.

Publishing packages

Prepare a .period file with export statements and run:

period publish greet.period --version 1.0.0 --registry-file registry.json

This computes the file checksum and updates registry.json. Next, upload the .period file to your release host (for example, as a GitHub Release asset) and deploy the updated registry.json to GitHub Pages. The default download URL assumes GitHub Releases at https://github.com/period-lang/registry/releases/download/{name}-{version}/{name}-{version}.period; override it with --base-url if you use Cloudflare R2 or another host.

Private or custom registries

Point the CLI at a different registry with the PERIOD_REGISTRY environment variable:

PERIOD_REGISTRY=https://my-org.github.io/registry period install

Built-in Functions

Period provides a small set of built-in functions. They are called like any other function using with.

show <value>

Prints a value followed by a newline. Showing an empty string prints a blank line.

show "hello".  -- hello
show "".       -- (blank line)

length with <value>

Returns the number of characters in a string, elements in a list, or keys in a dictionary.

show length with "hello".  -- 5
show length with [1, 2, 3].  -- 3

string with <value>

Converts a value to its string representation.

show string with 42.  -- "42"

number with <value>

Parses a string or converts a value to a number. Booleans become 1 or 0.

show number with "3.14".  -- 3.14

integer with <value>

Converts a value to an integer. Strings are parsed; numbers are truncated toward zero; booleans become 1 or 0.

show integer with "42".  -- 42
show integer with 3.14.  -- 3

boolean with <value>

Converts a value to a boolean. Zero, empty collections, empty strings, and nothing become false; everything else becomes true. This is an explicit conversion, not an implicit truthiness rule: conditions in if and while still require a boolean.

show boolean with 1.  -- true
show boolean with "".  -- false

type with <value>

Returns the type name as a string: nothing, boolean, integer, number, string, list, dictionary, function, built-in, class, error, or instance of <ClassName>.

show type with [1, 2].  -- "list"

input

Reads one line from standard input and returns it as a string.

show "What is your name?".
let name be input.
show "Nice to meet you, " + name + ".".

error with <message>

Raises a runtime error with the given message. Useful inside the test module or for explicit precondition checks.

define require with boolean condition:
    if not condition, then:
        error with "precondition failed".

File I/O

Use read to load a file's contents into a variable, and write to save a string to a file. Paths are relative to the source file's directory.

read content from "input.txt".
write content to "output.txt".

If the file cannot be read or written, the program stops with a runtime error.

Exceptions

Use try and catch to handle runtime errors without crashing. The catch variable receives an error object with message, line, and col properties.

try:
    let x be 1 / 0.
catch err:
    show the message of err.
    show the line of err.

Only runtime errors are caught; normal return statements still propagate out of the try block.

Exports

By default, all top-level names defined in a module are visible to importers. Use export to restrict the public API:

define helper with x:
    return x * 2.

define publicFn with x:
    return helper with x.

export publicFn.

When a module contains an export statement, only the listed names are imported. Built-in modules always export their documented names.

REPL

Running period without arguments starts the interactive Read-Eval-Print Loop. Type statements ending with a period and press Enter. Multi-line blocks use the ... prompt until the final line ends with a period.

$ period
Period REPL. Type 'exit.' or 'quit.' to leave, or Ctrl+C.

>>> let x be 10.
>>> show x * 2.
20
>>> if x == 10, then:
...   show "ten".
ten
>>> exit.

Use exit., quit., or Ctrl+C to leave the REPL.

VS Code Extension

The Windows installer automatically installs the Period VS Code extension. If you need to install it manually:

code --install-extension period-language.vsix

The extension provides:

  • Syntax highlighting for .period files
  • LSP-powered diagnostics with red/yellow squiggles
  • Hover information for keywords, built-ins, and user-defined symbols
  • Auto-completion for keywords, built-ins, and defined names
  • Go to Definition for variables, functions, classes, methods, and parameters

If the language server is not found automatically, set period.languageServerPath in VS Code settings to the full path of period.exe.

Error Messages

Period reports errors with the exact line and column range. Semantic and type errors are collected and reported together; parse errors currently stop at the first problem.

example.period:1:11: error: Expected '.' at the end of a 'let' statement.
       1 | let x be 5
         |           ^

The static checker also verifies that functions and methods with a declared return type return a value on every control-flow path, and that imported local modules actually exist:

example.period:3:1: error: function 'f' may not return a value on all paths
    3 | define f returns number:
      | ^

main.period:1:1: error: module not found './helper'
    1 | import ./helper.
      |       ^

Runtime errors, such as division by zero or undefined variables, also print the offending source line with a caret pointing to the error, similar to Python:

example.period:2:8: runtime error: Division by zero.
    2 | show 0 / 0.
      |        ^

All diagnostics come from the single Period frontend, so error locations and messages are consistent whether you run a file or use the LSP.

Grammar Reference

A concise reference for the Period language grammar.

program        := <statement>*
statement      := "let" (<type>)? <identifier> "be" (<type>)? <expression> "."
                | "set" <expression> "to" <expression> "."
                | "show" <expression> "."
                | "read" <identifier> "from" <expression> "."
                | "write" <expression> "to" <expression> "."
                | "try" ":" <block> "catch" <identifier> ":" <block>
                | "export" <identifier> ("," <identifier>)* ("and" <identifier>)? "."
                | "if" <expression> (",")? "then" ":" <block> (("otherwise" | "else") ":" <block>)?
                | "while" <expression> "repeat" ":" <block>
                | "for" <identifier> "in" <expression> "repeat" ":" <block>
                | "define" <identifier> ("with" <params>)? ("returns" <type>)? ":" <block>
                | "class" <identifier> ":" <class-body>
                | "return" <expression>? "."
                | "import" <module-path> ("," <module-path>)* ("and" <module-path>)? "."
                | <expression> "."
block          := <indent> <statement>+ <dedent>
class-body     := <indent> (<init> | <method>)+ <dedent>
init           := "init" ("with" <params>)? ":" <block>
method         := "define" <identifier> ("with" <params>)? ("returns" <type>)? ":" <block>
module-path    := <identifier> | "./" <path> | "../" <path>
arguments      := <expression> ("," <expression>)*
type           := <type-name> | "list" "of" <type> | "dictionary" "of" <type> "to" <type>
                | <type> (("or" | ",") <type>)+
type-name      := "nothing" | "boolean" | "integer" | "number" | "string" | "list" | "dictionary" | "function" | "class" | <identifier>
expression     := <or-expr>
or-expr        := <and-expr> ("or" <and-expr>)*
and-expr       := <not-expr> ("and" <not-expr>)*
not-expr       := "not" <not-expr> | <comparison>
comparison     := <additive> (("=="|"!="|"<"|">"|"<="|">=") <additive>)?
additive       := <multiplicative> (("+"|"-") <multiplicative>)*
multiplicative := <power> (("*"|"/"|"%") <power>)*
power          := <unary> ("**" <power>)?
unary          := ("-"|"not") <unary> | <call>
call           := <primary> ("from" <module-path>)? ("with" <arguments>)? ("[" <expression> "]")*
primary        := <number> | <string> | <bool> | "nothing" | "input" | "this"
                | <identifier> | "(" <expression> ")" | "[" <arguments>? "]" | "{" <pairs>? "}"
                | "new" <primary> ("with" <arguments>)?
                | "tell" <expression> "to" <identifier> ("with" <arguments>)?
                | "the" <identifier> "of" <primary>

Tips & Best Practices

  • End every statement with a period. Missing periods are the most common parse error.
  • Use indentation for blocks. After if ... then:, while ... repeat:, for ... in ... repeat:, define ... :, class ... :, or init ... :, indent the body by one level (four spaces is recommended).
  • Prefer functions and methods over deep nesting. They help keep programs article-like.
  • Use comments as paragraphs. A short comment before each section makes the code read like a document.
  • Check error locations. Diagnostics point to the exact character range, so read them carefully.
